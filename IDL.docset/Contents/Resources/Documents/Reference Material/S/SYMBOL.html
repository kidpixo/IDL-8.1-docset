SYMBOL

The SYMBOL function adds a symbol (with an optional text label) to the current IDL graphic at the x,y location.

Example

This example draws a map of Colorado along with symbols and names for two of its cities.

Note: The symbol location is specified with lat/lon coordinates, and is automatically transformed to the data space.

m = MAP('orthographic', $

  LIMIT=[35.86, -110.51, 42.58, -100.01], $

  CENTER_LONGITUDE=-105, CENTER_LATITUDE=40)

c = MAPCONTINENTS(/USA, /HIRES)

 

grid = m.MAPGRID

grid.HIDE = 1

 

s1 = SYMBOL(-104.98, 39.74, 'Star', /SYM_FILLED, $

  SYM_COLOR='red', LABEL_STRING='Boulder', /DATA)

s2 = SYMBOL(-107.82, 37.94, 'Star', /SYM_FILLED, $

  SYM_COLOR='red', LABEL_STRING='Telluride', /DATA)

Additional Examples

See Additional Examples for more examples using the SYMBOL function.

Syntax

Result = SYMBOL( X, Y, Symbol)

Note: To resize or scale a symbol, use the SYM_SIZE or SYM_FONT_SIZE properties rather than the SCALE method.

Return Value

Returns a reference to the created symbol. Use the reference to manipulate the symbol after creation by changing properties or calling methods.

Arguments

X

The scalar value describing the x location where the symbol will be placed. The x coordinate may be specified in data, normal (the default), or device coordinates.

Y

The scalar value describing the y location where the symbol will be placed. The y coordinate may be specified in data, normal (the default), or device coordinates.

Symbol

A string giving the symbol name. See Formatting IDL Graphics Symbols and Linesfor a list of valid symbol strings.

If SYM_FONT_NAME is set, this string is displayed in that font.

Use the Symbol argument when creating a graphic symbol. Use the SYMBOL property to modify a previously-created symbol.

Note: Symbol names of more than one character are case insensitive, while single-character names are case sensitive.

Properties

DATA (Init)

Set this property to 1 if the input arguments are specified in data coordinates. Setting this property inserts the symbol into the data space. Otherwise, the symbol is added to the annotation layer.

If the data space has a map projection, the X and Y arguments are automatically transformed to the projection.

DEVICE (Init)

Set this property if the X and Y coordinates are specified in device coordinates (pixels).

LABEL_COLOR

Set this property to a string or RGB vector giving the text color. The default value is 'black'.

LABEL_FILL_BACKGROUND

Set this property to 1 to fill the area inside the text box.

LABEL_FILL_COLOR

Set this property to the fill color inside the text box.

LABEL_FONT_NAME

Set this property to a string specifying the name of the IDL or system font to use for the text string. The default value is 'Helvetica'.

LABEL_FONT_SIZE

Set this property to an integer or floating-point value giving the font size in points. The default value is 16.

LABEL_FONT_STYLE

Set this property to an integer or string specifying the font style to be used for the text string. Allowed values are:

Set this property to a 2-element array specifying an amount to shift the location of the label in the X and Y directions. If the symbol is in the data space, these values should be specified using data coordinates. If the symbol is in the annotation layer, these values should be specified in normal coordinates.

LABEL_STRING

Set this property to a string or an array of strings containing the label text. If LABEL_STRING is an array of strings, each string is placed on a separate line.

LABEL_TRANSPARENCY

Set this property to an integer between 0 and 100 that specifies the percent transparency of the text. The default value is 0.

NORMAL (Init)

Set this property if the input arguments are specified in normalized (0, 1) coordinates (the default coordinate system).

SYM_COLOR

Set this property to the symbol color. See Formatting IDL Graphics Symbols and Linesfor a list of the valid color strings.

SYM_FILLED

Set this property to fill the symbols.

SYM_FILL_COLOR

Set this property to the color of the filled portion of the symbol. See Formatting IDL Graphics Symbols and Linesfor a list of valid color strings.

If this property is not set, the symbol fill color will match the SYM_COLOR.

SYM_FONT_FILL_BACKGROUND

Set this property to 1 to fill the area inside the text box when using a text string as the symbol. If the text symbol is not displayed, this property is ignored.

SYM_FONT_FILL_COLOR

Set this property to the color inside the text box. If the text symbol is not displayed, this property is ignored. See Formatting IDL Graphics Symbols and Linesfor a list of the valid color strings.

SYM_FONT_NAME

Set this property to a string specifying the name of the IDL or system font to use for the text symbol. The default value is the NULL string. If this property is specified, the value of Symbol should be set to a string that serves as the symbol when displayed in the specified font. Setting this value automatically displays the text symbol. Set this value to a NULL string ('') to hide the text symbol.

SYM_FONT_SIZE

Set this property to an integer or floating-point value giving the font size (in points). The default value is 12. If the text symbol is not displayed, this property is ignored.

SYM_FONT_STYLE

Set this property to an integer specifying the angle (in degrees) to rotate the symbol only (not the text and symbol together).

SYM_SIZE

Set this property to a floating point value specifying the plot symbol size. The default value is 1.

SYM_THICK

Set this property to a floating point value between 0 and 10 that specifies the thickness (in points) of the symbol. A value less than 1 displays as one pixel on the screen, but prints as a very thin line.

SYM_TRANSPARENCY

Set this property to an integer between 0 and 100 that specifies the percent transparency of the symbols. The default value is 0.

SYMBOL

Set this property to a string giving the symbol name. See Formatting IDL Graphics Symbols and Linesfor a list of valid symbol strings.

If SYM_FONT_NAME is set, set this property to a string that is displayed in that font.

Use the Symbol argument when creating a graphic symbol. Use the SYMBOL property to modify a previously-created symbol.

Note: Symbol names of more than one character are case insensitive, while single-character names are case sensitive.

TARGET (Init)

If points are specified in data coordinates, set this property to the graphic object in which to insert the symbol. By default, the current graphic is used.

UVALUE

Set this property to an IDL variable of any data type.

WINDOW (Get Only)

This property retrieves a reference to the WINDOW object which contains the graphic.

Additional Examples

Add a symbol to the data space of a plot

p = PLOT(/TEST)

s = SYMBOL(50, 0, '*', SYM_COLOR='Blue', SYM_SIZE=2, $

  SYM_THICK=3, /DATA)

Add a symbol to the annotation layer of a plot

p = PLOT(/TEST)

s = SYMBOL(0.5, 0.5, '*', SYM_COLOR='Blue', SYM_SIZE=2, $

  SYM_THICK=3, /NORMAL)

Add a character symbol to a map

In the Symbol font, the characters 'a' and 'b' are rendered as the alpha and beta symbols.

m = MAP('orthographic', $

  LIMIT=[35.86, -110.51, 42.58, -100.01], $

  CENTER_LONGITUDE=-105, CENTER_LATITUDE=40)

c = MAPCONTINENTS(/USA, /HIRES)

grid = m.MAPGRID

 

grid.HIDE = 1

s1 = SYMBOL(-104.48, 39.58, 'a', $

  SYM_COLOR='Green', LABEL_STRING='Primary Site', $

  /DATA, LABEL_POSITION='Top', $

  SYM_FONT_NAME='symbol', SYM_FONT_SIZE=26)

 

s2 = SYMBOL(-106.21, 39.38, 'b', $

  SYM_COLOR='Green', LABEL_STRING='Backup Site', $

  /DATA, LABEL_POSITION='Bottom', $

  SYM_FONT_NAME='symbol', SYM_FONT_SIZE=26)

Add a Unicode character symbol to a plot

For an explanation of the !Z syntax for choosing a Unicode character, refer to Positioning Commands.

Note: The Arial Unicode MS font used in this example may not be loaded on your system, but the syntax for choosing a Unicode character is the same for all platforms.

p = PLOT( FINDGEN(10), XSTYLE=1, YSTYLE=1)

s2 = SYMBOL( 0.3, 0.7, '!Z(7546)', /NORMAL, $

  LABEL_STRING='Unicode Symbol', SYM_FONT_SIZE=20, $

  SYM_FONT_NAME='Arial Unicode MS')
